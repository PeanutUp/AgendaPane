# Security policy

## Supported versions

Security fixes are provided for the latest released version of AgendaPane.

## Reporting a vulnerability

Please do not publish sensitive vulnerability details in a public issue.

Use GitHub's **Security → Report a vulnerability** option in the [AgendaPane repository](https://github.com/PeanutUp/AgendaPane/security) when available. If private reporting is unavailable, open a short issue requesting a private contact channel without including exploit details.

For ordinary bugs that do not expose user data or create a security risk, use [GitHub Issues](https://github.com/PeanutUp/AgendaPane/issues).

Please include the AgendaPane version, Obsidian version, operating system, impact, reproduction steps, and any suggested mitigation. Reports will be acknowledged as soon as practical.
